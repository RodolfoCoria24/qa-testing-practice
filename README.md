# 🧪 QA Testing Practice – Rodolfo Coria

Este repositorio contiene prácticas básicas de Aseguramiento de Calidad (QA), enfocadas en pruebas manuales sobre aplicaciones web.

---

## 🎯 Objetivo
Aplicar conceptos fundamentales de testing para validar funcionalidades, identificar errores (bugs) y mejorar la calidad del software.

---

## 🛠️ Herramientas utilizadas
- Postman (básico)
- Navegador web (Chrome)
- Pruebas manuales

---

## ✅ Casos de Prueba

### 🔐 Caso 1: Login exitoso
**Entrada:**
- Usuario: correcto
- Contraseña: correcta  

**Resultado esperado:**
- El usuario accede al sistema correctamente  

**Resultado obtenido:**
- Funciona correctamente  

---

### ❌ Caso 2: Login con datos incorrectos
**Entrada:**
- Usuario: incorrecto
- Contraseña: incorrecta  

**Resultado esperado:**
- Mostrar mensaje de error  

**Resultado obtenido:**
- Muestra mensaje de error correctamente  

---

### 📩 Caso 3: Validación de formulario
**Entrada:**
- Campos vacíos  

**Resultado esperado:**
- Mostrar validación de campos obligatorios  

**Resultado obtenido:**
- Algunos campos no muestran validación  

---

## 🐞 Reporte de Bugs

### Bug 1: Botón no responde en mobile
**Descripción:**
El botón de contacto no funciona correctamente en pantallas pequeñas.

**Severidad:** Media  
**Prioridad:** Alta  

---

### Bug 2: Falta validación en formulario
**Descripción:**
El campo de email permite ingresar datos sin formato válido.

**Severidad:** Alta  
**Prioridad:** Alta  

---

### Bug 3: Diseño desalineado
**Descripción:**
Elementos visuales se desordenan en resoluciones menores a 768px.

**Severidad:** Baja  
**Prioridad:** Media
